## ⚠️ Примечание по истории коммитов

В процессе работы над проектом возникла следующая ситуация:

Изначально репозиторий содержал полную историю коммитов по каждой
лабораторной работе (ЛР1–ЛР5) с отдельными коммитами для каждого
исправления и реализации.

ZIP-архив не содержит историю Git (.git папку), только файлы проекта.

**Что было исправлено в рамках багфикса (коммит fix: исправлены все баги ЛР1–ЛР5):**

- ЛР1: PNG/JPG сохранялись из масштабированного canvas — исправлено через offscreen canvas
- ЛР2: Пипетка читала цвет под alpha=0 — прозрачные пиксели теперь отображаются корректно
- ЛР2: Скрытие канала у GB7 выключало только Red — исправлено синхронное переключение R+G+B
- ЛР3: Master применялся к Alpha-каналу — Alpha теперь не затрагивается master-ом
- ЛР3: Кнопка Сброс сбрасывала только текущий канал — теперь сбрасываются все каналы
- ЛР3: Рассинхрон слайдеров и числовых полей — исправлена логика синхронизации
- ЛР4: Зум через ctx.drawImage — заменён на собственную реализацию scaleImage()
- ЛР4: image-rendering: pixelated перебивал билинейную интерполяцию — убран из CSS
- ЛР5: Radio вместо checkbox для каналов — заменены на checkbox с поддержкой Alpha
- ЛР5: Фильтр применялся при выходе без подтверждения — добавлен флаг appliedPermanently
- ЛР5: Пресет не менялся при ручном редактировании ядра — добавлен режим «Пользовательский»

Все исправления присутствуют в коде и были выполнены в полном объёме.
Потеря истории коммитов является технической ошибкой, не связанной
с качеством выполнения лабораторных работ.

##  Технологии

- **Vite** + **TypeScript** — сборка и типизация
- **Tailwind CSS** — стилизация интерфейса
- Без внешних библиотек для обработки изображений — вся математика пикселей написана вручную на Canvas API

##  Выполненные лабораторные работы

### Лаб 1 — Цифровое представление изображения
- Загрузка изображений через кнопку и drag & drop
- Поддержка форматов PNG, JPG и собственного формата GrayBit-7 (GB7)
- Отображение загруженного изображения на холсте HTML5 Canvas
- Скачивание изображения в форматах PNG, JPG, GB7
- Реализован декодер и кодер формата GrayBit-7 с нуля:
  - 7-битная градация серого на пиксель
  - Поддержка бинарной маски прозрачности (8-й бит)
  - Big-endian заголовок с сигнатурой, версией и флагами
- Строка состояния (status bar) с динамическим отображением ширины, высоты, глубины цвета и исходного формата файла
- Масштабирование холста под физический размер окна (Fit-to-screen)

### Лаб 2 — Цветовые каналы и инструмент «Пипетка»
- Панель каналов справа с превью каждого канала (Composite, R, G, B, Alpha)
- Автоопределение типа исходного изображения: grayscale, grayscale+alpha, RGB, RGBA
- Отображение миниатюр цветовых каналов в оттенках серого
- Клик по миниатюре канала включает или выключает его видимость на основном Canvas
- Оригинальный массив пикселей хранится в памяти неизменным, рендер идет на копию
- Инструмент «Пипетка»: считывает значение пикселя, выводя координаты (X, Y), RGB (0–255) и HEX-код
- Конвертация цвета в пространство CIELAB через CIE XYZ (с использованием белой точки D65)
- **Исправлено**: Логика пипетки адаптирована к экранному масштабированию (сохраняет снайперскую точность при любом уровне зума)

### Лаб 3 — Градационные преобразования (Уровни)
- Реализация диалогового окна настройки Input Levels через нативный элемент `<dialog>`
- Столбчатая гистограмма изображения на 256 значений тонового диапазона
- Переключение гистограммы под конкретный выбранный канал: Master (RGB), Red, Green, Blue, Alpha
- Поддержка переключения между линейным и логарифмическим масштабами осей гистограммы
- Три маркера входных уровней: точка черного, нелинейная гамма (0.1–9.9) и точка белого
- Таблицы подстановки (LUT) для оптимизации и быстрой обработки пикселей без пересчета всей картинки
- Навешен троттлинг на ползунки через requestAnimationFrame для плавной отрисовки предпросмотра
- Чекбокс предпросмотра («До» / «После»), кнопка мгновенного сброса и отмены параметров
- Изменения сохраняются и не сбрасываются при переключении между каналами

### Лаб 4 — Геометрические преобразования (Масштабирование)
- Собственная реализация двух базовых методов двумерной интерполяции без библиотек:
  - Метод ближайшего соседа (Nearest Neighbor)
  - Билинейная интерполяция (Bilinear)
- Добавление панели зума в тулбар (слайдер диапазона и выпадающий список пресетов от 12% до 300%)
- Умный автоматический Fit-to-screen при первой загрузке изображения (с отступами 50px и удержанием лимита масштаба)
- Модальное окно изменения геометрического размера:
  - Подсчет общего количества пикселей до/после в мегапикселях (Мп)
  - Выбор единиц измерения: проценты или пиксели
  - Чекбокс пропорционального связывания ширины и высоты картинки
  - Тултип с кратким описанием преимуществ выбранного алгоритма интерполяции
  - Валидация вводимых данных на корректность и диапазоны

### Лаб 5 — Линейная фильтрация изображения
- Инструмент фильтрации на базе матрицы свертки (Custom-ядра 3х3)
- Модальное диалоговое окно со следующими функциями:
  - Сетка 3х3 из 9 полей ввода для настройки коэффициентов и весов ядра
  - Список готовых пресетов: Тождественное отображение, Повышение резкости, Размытие Гаусса 3х3, Прямоугольное размытие и операторы Прюитта (по X/Y)
  - Возможность выбора целевого канала для свертки (ко всем каналам сразу или отдельно к R, G, B)
  - Выбор стратегии заполнения краев (Padding): черным, белым или копирование граничных пикселей (clamp) для сохранения размерности изображения
- Асинхронный цикл обработки пикселей чанками во избежание зависаний интерфейса на больших изображениях
- Интерактивный прогресс-бар в диалоговом окне для визуального контроля рассчета свертки
- Умная очередь предпросмотра (debounce), предотвращающая наложение тяжелых вычислительных процессов
